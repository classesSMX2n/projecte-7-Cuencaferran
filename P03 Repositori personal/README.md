# 📌 Lliurament del projecte – GitHub i control de versions

## Introducció

L’ús de **GitHub** i el control de versions mitjançant **git** ja és una tècnica que heu d’haver interioritzat. Per tant, no us ha d’estranyar que el repositori de GitHub sigui la **base del lliurament del projecte**.

A la tasca inicial del projecte, vau crear el vostre repositori a partir del repositori base proporcionat. Ara toca comprovar que el teniu **llest per al lliurament**.

## Estructura del repositori

Recordeu que cada **projecte** i cada **producte** ha de tenir la seva carpeta amb un criteri clar, per exemple:

- `T01`, `T02`, etc. – per a tasques
- `P01`, `P02`, etc. – per a productes o lliuraments

### Dins de cada carpeta

| Fitxer | Descripció |
|--------|-------------|
| `README.md` | Descripció de l’activitat (pot ser l’enunciat resumit). Format **Markdown** i estructurat en seccions. |
| `solucio.md` | Solució de l’activitat: codi, enllaços, explicacions tècniques, etc. |
| `pics/` (carpeta) | Totes les imatges necessàries per a la documentació. |

> ⚠️ **Important:** El nom dels fitxers ha de ser exactament `README.md` i `solucio.md`. Respecteu les majúscules/minúscules.

## Què cal lliurar

✅ **Repositori assignat al GitHub Classroom**  
✅ No s’accepten lliuraments en repositoris diferents de l’assignat a GitHub Classroom.

## Comprovació final

Abans de lliurar, verifiqueu:

- [ ] El repositori té l’estructura de carpetes `Txx` / `Pxx`.
- [ ] Cada carpeta conté `README.md`, `solucio.md` i (si escau) `pics/`.
- [ ] El `README.md` de l’arrel conté aquesta explicació (o la que correspongui).
- [ ] Heu fet `git add`, `git commit` i `git push` a la branca principal.
